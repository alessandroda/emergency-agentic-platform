# Hermes Agent Rules

## Project
Operational atmospheric emergency response platform.

## Architecture
- Backend: FastAPI
- Frontend: Leaflet
- Dispersion: OpenDrift
- Meteorology: GFS

## Constraints
- No database
- No microservices
- No Docker initially
- No authentication
- No enterprise architecture
- Keep implementation minimal

## Coding Rules
- One task at a time
- Modify only requested files
- Avoid unnecessary dependencies
- Preserve working functionality
- Prefer simple explicit code
- Avoid overengineering

## Workspace
All files MUST remain inside this repository.
Never write files outside the repository root.
